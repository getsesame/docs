Run everything below in your OpenClaw container's **App terminal** (hPanel → your app → **App terminal**), as root. The gateway runs as the `node` user (`HOME=/data`), so Sesame commands are prefixed with `runuser -u node --`. Installing into `/data` and wrapping via the compose file keeps the setup alive across restarts, reboots, and container recreates — only `/data` and the compose file survive a recreate, and both hold everything Sesame needs.

Requires the **cloud broker** at [getsesame.dev](https://getsesame.dev). See the [main OpenClaw guide](/agents/openclaw) for the concepts (secrets, injection, approvals).

## 1. Add the secret (browser)

In [getsesame.dev](https://getsesame.dev) → **Secrets → Add**, add one secret for the host OpenClaw calls — usually `api.openai.com` (Bearer). Then set an **auto-approve policy** on that host so chat doesn't pause for approval every turn.

## 2. Install + register (container shell)

```bash
# install Sesame into the persistent volume, fix permissions, create the runtime dir
mkdir -p /data/.local/bin && curl -fsSL https://getsesame.dev/install.sh | sh -s -- --prefix /data/.local/bin
chown -R node:node /data/.local /data/.config
mkdir -p /run/user/1000 && chown node:node /run/user/1000 && chmod 700 /run/user/1000

# edge proxy
runuser -u node -- /data/.local/bin/sesame proxyd install

# register this device — open the printed URL in your browser and approve
runuser -u node -- /data/.local/bin/sesame login

# pre-check: must print SESAME_OK before you wrap anything
runuser -u node -- /data/.local/bin/sesame launch -- echo SESAME_OK
```

## 3. Wrap the gateway (container shell)

```bash
cat > /data/sesame-boot.sh <<'EOF'
#!/bin/bash
set -e
chown -R node:node /data
find /data/.openclaw/agents -name '*.lock' -delete 2>/dev/null || true
mkdir -p /run/user/1000 && chown node:node /run/user/1000 && chmod 700 /run/user/1000
cd /hostinger
exec runuser -u node -- /data/.local/bin/sesame launch -- "$@"
EOF
chmod +x /data/sesame-boot.sh
```

## 4. Point the compose file at it (host shell or panel YAML editor)

In `docker-compose.yml`, under the `openclaw` service, add both keys (setting `entrypoint` clears the image's default command, so restate it):

```yaml
    entrypoint: ["/data/sesame-boot.sh"]
    command: ["node", "server.mjs"]
```

Apply:

```bash
cd /docker/<app> && docker compose up -d --force-recreate
```

## 5. Verify

```bash
runuser -u node -- /data/.local/bin/sesame launch -- \
  curl -s -o /dev/null -w '%{http_code}\n' https://api.openai.com/v1/models   # → 200
```

Then send a chat in OpenClaw and approve the first call. The reply arrives with your real key injected server-side — OpenClaw never sees it.

## Run this with an AI agent

Paste this to an agent working in the container's App terminal. It runs the same steps and verifies them; it pauses at each 👤 for you.

```
Onboard this Hostinger OpenClaw onto Sesame (broker: getsesame.dev). Run in the container's App
terminal as root. The gateway runs as user `node` (HOME=/data), so prefix Sesame commands with
`runuser -u node --`. Install into /data and wrap via the compose file so it survives recreates.
Stop and ask me at each 👤.

1. Confirm a secret exists in getsesame.dev for the host OpenClaw calls (usually api.openai.com,
   Bearer), with an auto-approve policy. 👤
2. Install + fix dirs:
     mkdir -p /data/.local/bin && curl -fsSL https://getsesame.dev/install.sh | sh -s -- --prefix /data/.local/bin
     chown -R node:node /data/.local /data/.config
     mkdir -p /run/user/1000 && chown node:node /run/user/1000 && chmod 700 /run/user/1000
   Require sesame >= v0.3.71.
3. runuser -u node -- /data/.local/bin/sesame proxyd install
4. runuser -u node -- /data/.local/bin/sesame login  → give me the URL, wait for approval. 👤
5. Pre-check (must print SESAME_OK, else stop and fix login):
     runuser -u node -- /data/.local/bin/sesame launch -- echo SESAME_OK
6. Write /data/sesame-boot.sh replicating the STOCK /entrypoint.sh setup, then
   `exec runuser -u node -- /data/.local/bin/sesame launch -- "$@"`; chmod +x. In the host
   docker-compose.yml add under the service: entrypoint: ["/data/sesame-boot.sh"] and
   command: ["node","server.mjs"]. Then: docker compose up -d --force-recreate.
7. Verify and report ✅/❌:
     - the gateway process is a child of `sesame launch` and `sesame-proxyd` is running;
     - the gateway's /proc/<pid>/environ has HTTPS_PROXY + NODE_EXTRA_CA_CERTS and NO
       LD_LIBRARY_PATH (read it as `docker exec -u node`, not plain exec);
     - runuser -u node -- /data/.local/bin/sesame launch -- curl -s -o /dev/null -w '%{http_code}'
       https://api.openai.com/v1/models  returns 200.
8. Ask me to send a chat and approve the first call. If it returns {"detail":"Bad Request"}, a
   brokered OAuth host (e.g. chatgpt.com/codex) is overriding the agent's auth — broker
   api.openai.com and use provider: openai instead.
```
